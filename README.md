# KiC Practice — Kanji in Context Sentence Trainer

A self-hosted Japanese sentence practice app built around the
**Kanji in Context (Revised Edition)** textbook series. Sentences are
sourced from the [Tatoeba](https://tatoeba.org) corpus, filtered to match
exactly the kanji you have studied so far, and annotated with
authoritative furigana from Tatoeba's bracket-form transcriptions
(restricted to a curated whitelist of trusted contributors). Translation
quality is graded by Claude (Anthropic API); vocabulary is scored
locally against the contextual readings baked into each sentence.

## How it works

### The Anki deck

The source vocabulary comes from the publicly available Anki deck
**"Kanji in Context: Revised Edition"** (AnkiWeb ID 1996057559).
The deck covers all 2,136 Jōyō kanji organised into 156 lessons across
7 sections (KC1–KC7) and 3 stages.

The original deck has ~27% of cards missing their lesson tag (L001–L156).
The `augment_anki.py` script repairs this using two strategies:

1. **Rule-based inference** — a card's lesson is inferred as the latest
   lesson among all kanji numbers in its tags. Since KiC teaches kanji
   sequentially, a compound word can only appear once all its constituent
   kanji have been introduced.

2. **Manual overrides** — 20 kanji numbers whose lesson could not be
   inferred from the deck data alone were verified against the physical
   book and hardcoded in the script.

All augmentations are fully reproducible from the script with no
additional manual steps.

### Sentence filtering

For each Tatoeba sentence with a transcription from a trusted author, the
build script computes a **catalog lesson** L and writes the sentence under
`sentences/gradeN/<short|medium|long>/L.json` if:

- **Every kanji in the sentence** is either base-grade (Grade 1 or Grade 1+2,
  per the chosen base) **or** in KiC. Sentences containing kanji outside both
  sets are dropped.
- **At least one matched KiC vocabulary word from the catalog lesson** is
  present in the sentence. This is what makes the sentence relevant practice
  for that lesson — sentences whose catalog lesson came purely from an
  unmatched non-base-grade kanji (with no KiC vocab match at that level)
  are discarded.
- **At most N current-lesson KiC vocabulary words** appear, where N is the
  `--max-required-words` cap (default 2). Combined with the lower bound
  above, every kept sentence has between 1 and N current-lesson KiC words.
- Sentences are bucketed by length into four buckets: very_short (≤5 chars),
  short (6–14 chars), medium (15–30 chars), and long (≥31 chars). There is
  no per-bucket cap — the requirement of a current-lesson KiC match acts as
  a natural cap on how many sentences each lesson collects.

A KiC vocabulary form only counts as a match if it covers an entire bracket
span in the transcription. A single-kanji form that matches only part of a
multi-kanji bracket (e.g. `本` within `[本当|ほん|とう]`) is rejected; the
bracket falls to `non_kic_words` instead. This ensures vocabulary quiz targets
correspond to whole transcribed units rather than sub-strings.

The catalog lesson L is the **highest** of:

- The latest KiC lesson among matched KiC vocabulary words in the sentence.
- The introduction lesson of the latest non-base-grade KiC kanji that appears
  in any unmatched bracket of the transcription.

This means a sentence whose KiC vocab is from L005 but which contains an
unmatched kanji introduced in L020 is filed under L020, not L005 — the
catalog lesson reflects the latest KiC content the learner needs to know
to make sense of the whole sentence. A sentence where no KiC vocabulary form
matches any bracket (e.g. because the compound it contains is not in the deck
as a standalone form) is dropped entirely.

Each sentence record contains:

- `id` — stable content hash for the app
- `ja_transcribed` — the bracket-form transcription (authoritative furigana); the plain surface sentence is derived from this at runtime
- `en` — list of English translations
- `source` — corpus the sentence came from (`tatoeba`, or the name given to `--corpus`)
- `lesson` — the catalog lesson tag
- `transcription_author` — the trusted Tatoeba contributor who wrote the brackets (null for supplemental corpora)
- `length_bucket`, `char_length`
- `required_words` — number of current-lesson KiC vocab matches (always ≥1)
- `kic_words_current_lesson`, `kic_words_previous_lessons` — matched KiC
  vocabulary, each with `word`, `lesson`, `kanji_ids`, `reading` (derived
  from the transcription), `quiz`
- `non_kic_words` — supplemental vocabulary derived from unmatched bracket
  units, each with `surface`, `reading`, `kanji_ids`, `max_kic_lesson`,
  `quiz`
- `target_kanji_ids` — KiC IDs of kanji in current-lesson matched vocab
- `sentence_kanji_ids` — KiC IDs of every recognised kanji in the sentence

### Furigana via Tatoeba transcriptions

Tatoeba publishes bracket-form furigana transcriptions of Japanese
sentences. The build script imports `data/jpn_transcriptions.tsv` and
discards anything whose transcription author is not in
`data/jpn_authors.tsv` — the curated whitelist of trusted contributors.
The resulting bracket string is stored verbatim in each sentence record
as `ja_transcribed` and rendered in the app as standard `<ruby><rt>`
markup at runtime.

Because the transcription identifies every word-like unit in the
sentence, the build can not only attribute the contextual reading of
each KiC vocabulary match but also identify every unmatched bracket
group as a "supplemental" vocabulary item. The app quizzes both axes:
matched KiC vocabulary and supplemental non-KiC vocabulary are scored
on separate tracks (KiC: 2 points each, non-KiC: 1 point each).

The app makes Claude calls **only for translation scoring** — never for
furigana or vocabulary lookup. Translation calls use Anthropic's Messages
API from the browser with the `anthropic-dangerous-direct-browser-access:
true` header. The current app model is `claude-sonnet-4-6`; if Anthropic
retires or restricts that model, update the `MODEL` constant in `index.html`.

### The practice app

A single `index.html` file. No build step, no framework dependencies.
Hosted on GitHub Pages, accessible from any browser including mobile.

**Features:**
- Configure current lesson range, base grade, and max target word count
- JP→EN practice mode with authoritative bracket-derived furigana
- One kana reading input per quiz target (KiC current/previous + Non-KiC)
- One English translation textarea
- Vocabulary scoring is forfeited if furigana is revealed before submission
- Translation scored by Claude (0–5) for meaning accuracy and completeness
- Per-axis score breakdown on each round (KiC / Non-KiC / Translation)
- Skipped sentences are listed on the session score screen
- Claude API key stored in browser `localStorage` — entered once per device,
  never in the codebase

## Setup

This setup is only needed for those who want to re-create the data files. If you just want to practice, this is not needed, just go ahead and run the `index.html` app since the data are already provided.

### Prerequisites

- Python 3.8+
- An Anki account (free) and the Anki desktop app (free for Mac/Windows)
- An Anthropic API account with an API key
- A GitHub account

### Step 1 — Get the Anki deck

1. Open Anki desktop
2. Go to **Tools → Add-ons** and make sure you are signed into AnkiWeb
3. Download deck ID **1996057559** from AnkiWeb
   (`File → Import` or browse to `ankiweb.net/shared/info/1996057559`)
4. The deck will appear as **"Kanji in Context: Revised Edition"**

### Step 2 — Export the original notes

1. In Anki, open **Browse**
2. Click your KiC deck in the left sidebar to select all its cards
3. Go to **Notes → Export Notes**
4. Choose: **Plain text**, separator **Tab**, tick **Include tags**
5. Save as `data/kic_original.txt` in this repo

### Step 3 — Augment the notes with missing lesson tags

```bash
cd scripts
python3 augment_anki.py \
  --input ../data/kic_original.txt \
  --output ../data/kic_augmented.txt
```

Expected output:
```
Records found: 9279
Kanji->lesson map: 2115 entries
Cards augmented: ~2508
Cards still unresolved: 0
```

### Step 4 — Re-import augmented notes into Anki (optional)

This step is only needed for your Anki study workflow. The sentence
pipeline uses `kic_augmented.txt` directly and does not require Anki.

1. In Anki: **File → Import**
2. Select `data/kic_augmented.txt`
3. Set **"Existing notes"** to **Update**
4. Import — only lesson tags are changed, nothing else

After importing, unsuspend cards lesson by lesson as you progress
through the book (Browse → filter by tag L001 → select all → unsuspend).

### Step 5 — Download the Tatoeba sentence pairs and transcriptions

Download the Japanese–English sentence pairs:

```
https://downloads.tatoeba.org/exports/per_language/jpn/jpn-eng_sentences.tsv.bz2
```

Decompress and save as `data/jpn_eng_sentences.tsv`.

Then download the Japanese transcriptions (bracket-form furigana):

```
https://downloads.tatoeba.org/exports/per_language/jpn/jpn_transcriptions.tsv.bz2
```

Decompress and save as `data/jpn_transcriptions.tsv`.

Both files are tens of megabytes and **not committed to the repo**. The
trusted-author whitelist (`data/jpn_authors.tsv`) is committed and used as-is.

### Step 6 — Build the sentence JSON files

```bash
cd scripts
python3 build_sentences.py \
  --tatoeba ../data/jpn_eng_sentences.tsv \
  --transcriptions ../data/jpn_transcriptions.tsv \
  --authors ../data/jpn_authors.tsv \
  --kic ../data/kic_augmented.txt \
  --corpus claude:../data/claude_supplement.jsonl \
  --output-dir ../sentences/grade2 \
  --base-grade 2 \
  --max-required-words 2
```

Repeat with `--base-grade 1 --output-dir ../sentences/grade1` for the
Grade 1 base set.

The `--corpus` flag is optional and repeatable. Each supplemental corpus is a
JSONL file where every line is `{"ja_transcribed": "...", "en": ["..."]}`.
Sentences from supplemental corpora pass through the same KiC matching and
lesson-cataloguing pipeline as Tatoeba sentences; the `source` field in each
output record identifies which corpus a sentence came from.

Each lesson produces four files (one per length bucket), e.g.
`sentences/grade2/very_short/L001.json`, `sentences/grade2/short/L001.json`,
`sentences/grade2/medium/L001.json`, and `sentences/grade2/long/L001.json`.

### Step 7 — Enable GitHub Pages

1. Push the repo to GitHub
2. Go to **Settings → Pages**
3. Set source to **Deploy from branch → main → / (root)**
4. Your app will be live at `https://yourusername.github.io/kic-practice/`

### Step 8 — Enter your API key

1. Open the app URL in your browser
2. Go to **Settings** (gear icon)
3. Paste your Anthropic API key — it is saved to `localStorage` and
   never leaves your browser

## Regenerating sentences

As you progress through the book you may want to regenerate sentences
with an expanded allowed set. Simply re-run Step 6. The script is
idempotent — it wipes everything under the chosen `--output-dir` before
writing the new bucketed files.

## Configuration reference

### `augment_anki.py`

| Argument | Default | Description |
|----------|---------|-------------|
| `--input` | required | Path to original Anki notes export |
| `--output` | required | Path for augmented output file |

### `build_sentences.py`

| Argument | Default | Description |
|----------|---------|-------------|
| `--tatoeba` | required | Path to Tatoeba JP-EN sentences TSV |
| `--transcriptions` | required | Path to Tatoeba bracket-form transcriptions TSV |
| `--authors` | required | Path to Tatoeba contributor whitelist TSV |
| `--kic` | required | Path to augmented KiC notes TSV |
| `--corpus` | _(none)_ | Extra corpus JSONL, `NAME:PATH` format; repeatable |
| `--output-dir` | `sentences` | Grade output root (e.g. `../sentences/grade2`) |
| `--base-grade` | 2 | Base kanji grade level (1 or 2) |
| `--max-required-words` | 2 | Max current-lesson KiC words per sentence (min is always 1) |

## Notes on deck quality

The source Anki deck (ID 1996057559) has known quality issues — some
cards have incorrect readings or meanings, and some vocabulary is
missing from certain chapters. Cross-reference with the physical book
when in doubt. The augmentation script addresses only the missing lesson
tags, not content errors.
